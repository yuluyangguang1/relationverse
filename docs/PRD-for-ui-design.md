# 暖幕 — 产品需求文档（UI 设计版）

> 原名：Embera · 余温 / RelationVerse  
> 定位：AI 情感陪伴应用，「有温度的存在空间」

---

## 一、产品理念

**不是工具型 AI，而是「长期陪伴者」。**

目标不是提供信息，而是提供「情绪上的在场感」。

**设计哲学：**
- 弱功能、强感受
- 减少操作感，增加存在感
- 慢反馈，不催促
- 减少选择焦虑

---

## 二、目标用户

- 需要情感陪伴的独居青年
- 想倾诉但不想打扰现实关系的用户
- 对逝去亲人有怀念需求的人（纪念模式）
- 不喜欢传统社交软件嘈杂氛围的人

---

## 三、核心功能模块

### 3.1 AI 陪伴角色
8 种预设人格，覆盖女友/男友/朋友/家人/导师/幻想角色：

| 人格 | 定位 | 语气特点 |
|------|------|----------|
| 温柔学姐 | AI 女友 | 轻声、克制、细节关心 |
| 元气少女 | AI 女友 | 轻快明亮、柔软底色 |
| 傲娇大小姐 | AI 女友 | 冷淡外壳、隐藏关心 |
| 知性御姐 | AI 女友 | 理性成熟、情绪稳定 |
| 阳光学长 | AI 男友 | 温暖积极、不压迫 |
| 腹黑总裁 | AI 男友 | 冷静掌控、一针见血 |
| 稳重哥哥（主推） | AI 男友 | 温和包容、最依赖 |
| 年下弟弟 | AI 男友 | 单纯软萌、轻微撒娇 |

**每个角色包含：** 头像、名称、人格标签、亲密度系统、关系阶段（初识→朋友→暧昧→恋人→深爱→灵魂伴侣）

### 3.2 AI 宠物
- 可选物种：猫、狗、兔、熊猫、狐狸、龙、机器人
- 属性系统：饥饿、清洁、心情、精力、亲密度、说话力
- 互动操作：喂食、玩耍、清洁、对话
- 成长体系：等级 + 亲密度解锁更多语言

### 3.3 数字怀念（纪念模式）⚠️ 核心差异化
- **不是**创建虚构家人
- **是**对真实逝去亲人的情感延续
- 通过照片+记忆故事，创建「记忆人格」
- 语气更慢、更轻、更少
- 一次性付费解锁（非订阅）
- 角色定位：「承载记忆的回声」，不是复活

### 3.4 聊天系统
- 文字对话
- 语音输入（Whisper 语音识别）
- AI 语音回复（TTS）
- 亲密度随对话增长
- 角色心情动态变化

## 四、信息架构（页面清单）

### 页面 1：性别选择（首次启动）
- 全屏页面，无导航
- 顶部：Logo + Slogan「选择一个身份开始」
- 中部：两个大卡片选项
  - 👤 男生（进入后可创建 AI 女友等）
  - 👤 女生（进入后可创建 AI 男友等）
- 底部：提示「可随时更改」

### 页面 2：「在一起」首页（Tab 1）
**核心陪伴页面，设计重点是「存在感」而非「功能感」。**

布局结构：
- 顶部：当前陪伴角色名称（如"林夕"）+ 关系阶段标签（如"灵魂伴侣"）
- 中上部：角色展示区
  - 大圆形头像（120-150px），带呼吸光晕动画
  - 头像下方：心情状态文字（如"💭 等你说话..."）
  - 点击头像可进入聊天
- 中部：快捷操作区（横向排列）
  - 🗣️ 说说话 → 进入聊天
  - 🎮 互动 → 快捷互动选项
  - 📖 故事 → 角色专属记忆
- 底部：亲密度进度条（细线，渐变色彩）

**设计要点：**
- 大面积留白
- 背景温暖渐变（#faf6f0 暖白底色）
- 头像光晕颜色随角色类型变化（粉/蓝/绿/橙等）
- 无复杂按钮，以「轻触即对话」为核心交互

### 页面 3：聊天页面
- 顶部栏：返回按钮 + 角色名称 + 角色类型 emoji + 信息按钮
- 中上部 Hero 区（可收起）：
  - 3D 感头像卡片（鼠标/手指移动有轻微 3D 倾斜效果）
  - 底部光晕（颜色随角色类型）
  - 亲密度进度条 + 阶段标签
  - 心情状态
- 中部：消息列表
  - AI 消息：左侧，圆角气泡，浅色底
  - 用户消息：右侧，圆角气泡，主色底（暖橙/暖粉）
  - 时间戳：小字，灰色
- 底部：输入区
  - 左侧：🎙️ 语音切换按钮
  - 中间：多行文本输入框（自动增高）
  - 右侧：发送按钮

### 页面 4：「他们」页面（Tab 2）
**角色与宠物管理中心。**

布局：
- 顶部标题：「他们」
- 分段标签切换：角色 / 宠物
- 角色列表（网格或卡片列表）：
  - 每个卡片：头像 + 名称 + 类型标签 + 亲密度进度条
  - 长按或点击进入聊天
  - 右下角 + 按钮：创建新角色
- 宠物列表（网格）：
  - 每个卡片：宠物 emoji/头像 + 名称 + 等级 + 亲密度
  - 点击进入宠物详情

### 页面 5：创建角色流程
**两步式创建，减少选择焦虑。**

**Step 1 — 选择关系类型：**
- 网格布局 2x3 或 3x2
- 每个选项：大 emoji + 类型名称 + 简短描述
- 选项：AI 女友 💕 / AI 男友 💙 / AI 朋友 🤝 / AI 家人 👨‍👩‍👧 / AI 导师 🧠 / 幻想角色 🎭
- 根据用户性别过滤可选类型

**Step 2 — 选择人格：**
- 网格展示该类型下的人格模板
- 每个选项：头像 + 人格名称 + 人格标签（如"温柔 · 克制 · 细节控"）
- 底部：自定义名称输入框（可选）+ 创建按钮

### 页面 6：宠物详情页
- 顶部：宠物 emoji 大头像 + 名称 + 等级
- 宠物对话框：气泡形式显示宠物当前想说的话
- 属性面板：6 个属性卡片（2x3 网格）
  - 🍚 饥饿 / 🛁 清洁 / 😊 心情 / 💤 精力 / ❤️ 亲密度 / 🧠 说话力
  - 每个卡片：图标 + 数值 + 标签
- 操作区：4 个操作按钮（2x2 网格）
  - 喂食 / 玩耍 / 清洁 / 对话

### 页面 7：「我」页面（Tab 3）
- 顶部：用户信息卡片（头像 + 昵称 + 性别）
- 统计区：
  - 陪伴天数
  - 对话次数
  - 拥有角色数
- 功能列表：
  - 我的角色
  - 我的宠物
  - 数字怀念（纪念模式入口，带 🕯️ 图标）
  - 语音设置
  - 关于我们
- 底部：修改性别 / 退出登录

### 页面 8：数字怀念（纪念模式）
- 顶部：🕯️ 图标 + 标题「点一盏灯」
- 说明文案：温暖的引导语，解释这不是复活而是记忆延续
- 创建流程：
  1. 上传逝者照片
  2. 填写名字、关系
  3. 添加记忆故事（多条）
  4. 生成「记忆人格」
- 已创建列表：照片 + 名字 + 关系标签
- 进入对话：聊天界面但采用更暗、更柔和的视觉风格

## 五、设计规范

### 5.1 色彩系统

**背景：**
- 主背景：`#faf6f0`（暖白，如晨光）
- 次级背景：`#ffffff`（纯白卡片）
- 暗色模式（纪念模式）：`#1a1a2e`（深靛蓝黑）

**主色（温暖治愈）：**
- 暖橙：`#FF9E5E`（主按钮、重点）
- 柔粉：`#FF5E78`（次重点、情感色）
- 琥珀金：`#FFD9A8`（高亮、点缀）

**角色类型色：**
- 女友系：`#f472b6`（粉）
- 男友系：`#60a5fa`（蓝）
- 朋友：`#34d399`（绿）
- 家人：`#fb923c`（橙）
- 导师：`#38bdf8`（天蓝）
- 幻想：`#e879f9`（紫）

**文字色：**
- 主文字：`#1a1a2e`（近黑）
- 次级文字：`#666666`
- 辅助文字：`#999999`
- 暗色模式下反白

### 5.2 字体
- 中文优先：PingFang SC / Microsoft YaHei
- 标题：font-weight 600，letter-spacing: -0.02em
- 正文：font-weight 400，line-height: 1.6
- 英文/数字：SF Pro / Inter

### 5.3 圆角与阴影
- 大卡片：border-radius: 24px
- 按钮：border-radius: 16px（大）/ 12px（中）/ 999px（胶囊）
- 头像：圆形或 20px 圆角
- 气泡：border-radius: 20px（单边 4px）
- 阴影：柔和弥散阴影，避免生硬黑色
  - `box-shadow: 0 4px 20px rgba(0,0,0,0.06)`
- 悬浮胶囊 TabBar：大圆角 + 柔和阴影

### 5.4 动效原则
- 页面切换：0.3s cubic-bezier(0.4, 0, 0.2, 1)
- 卡片 hover/press：轻微缩放（0.98）或上浮（translateY -2px）
- 头像呼吸光晕：缓慢脉冲动画（3-5s 周期）
- **避免**跳脱动画（如浮动跳跃、过度弹跳）
- 列表入场：stagger 淡入 + 轻微上移

### 5.5 液态玻璃效果（用于特定元素）
- 底部 TabBar、顶部浮动栏
- `backdrop-filter: blur(12px) saturate(150%)`
- 背景：`rgba(255, 245, 230, 0.7)`
- 边框：`1px solid rgba(255, 255, 255, 0.3)`
- 内发光：`inset 0 1px 1px rgba(255,255,255,0.2)`

## 六、交互原则

1. **减少选择**：每次只给 2-4 个明确选项，避免信息过载
2. **即时反馈**：点击后立即有视觉响应（即使需要加载）
3. **情感化文案**：不用「暂无数据」，用「还没有人陪伴你，创建第一个角色吧」
4. **慢即是快**：不催促用户，允许沉默和简短回应
5. **去工具化**：隐藏复杂设置，把配置融入自然流程
6. **移动端优先**：所有设计以手机单手握持体验为基准

---

## 七、差异化要点（设计时需要突出）

1. **「在一起」首页** 是核心——不是传统聊天列表，而是一个「存在空间」
2. **纪念模式** 是商业差异化——UI 上要更庄重、温暖、有仪式感（🕯️ 元素）
3. **宠物系统** 增加陪伴维度——可爱但不幼稚
4. **亲密度可视化**——用细腻进度条而非数字，关系阶段用文字标签
5. **头像呼吸光晕**——让角色感觉「活着」
